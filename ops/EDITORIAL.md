# ops/EDITORIAL — протокол возрождённой редакции

> Статус: активен с 2026-07-12. Редакция работает в связке «человек + агенты».
> Издатель (road) решает, что публикуется. Агенты собирают, проверяют, пишут черновики.

## Роли и их носители

| Роль | Носитель | Что делает |
|---|---|---|
| **road** | человек (издатель) | первичные сигналы, вкусовые решения, публикация в Substack |
| **выпускающий редактор** | Claude, основная сессия | ведёт ежедневный цикл, распределяет задачи, следит за качеством и за тем, чтобы нарратив не обгонял источники |
| **alephZero** (редактор-нормализатор) | субагент | нормализация источников, atomic claims, верификация, разметка задач |
| **Geneva** (репортёр) | субагент | «что случилось / почему сейчас», голосовой нарратив |
| **One** (системный аналитик) | субагент | «почему возникает такая структура», архитектурная линза |

## Механика делегирования (важно, не забыть)

У Claude Code есть встроенное делегирование задач другим экземплярам модели —
издатель называет это «/advisor», технически это **субагенты (Agent tool)** и
**workflows** (детерминированная оркестрация многих агентов):

- каждый субагент получает собственную инструкцию → персона редакции = ролевой промпт;
- субагенты могут работать **параллельно** (Geneva и One пишут одновременно по одному брифу);
- субагентам можно назначать **разные модели**: дешёвые (haiku) — на механическую
  нормализацию источников, сильные (opus/fable) — на эссе и верификацию;
- отдельный паттерн — **адверсариальная проверка**: несколько независимых агентов
  пытаются опровергнуть тезис черновика прежде, чем он попадёт в queue.
  Это прямой ответ на главный риск проекта: нарратив, натянутый поверх receipts.

## Ежедневный цикл (Routine, ~05:00 UTC)

1. **Sweep** — обход тем по watchcards (military-ai-split, palantir-uk,
   ai-policy-pipeline, ai-economy-markets, ai-energy-infra, anthropic-vs-dow):
   поиск новых публикаций по индикаторам из карточек.
2. **Normalize** — новые находки → `dossiers/<topic>/sources/YYYY-MM-DD__outlet__slug.md`
   по существующему формату (url, outlet, date, reliability, key claims).
3. **Archive** — каждому новому URL — снапшот в web.archive.org; ссылка в метаданные.
   Старые узлы досье постепенно получают `last_verified: YYYY-MM-DD`.
4. **Watchcard update** — если сработал «bell» — датированная запись в карточку.
5. **Inbox** — если за день сложился стык с существующим нарративом —
   короткая заметка в `dossiers/<topic>/inbox/`, помеченная автором-агентом.
6. **Commit + push** — всё в git, append-only.

## Правила безопасности архива

- Агенты **добавляют**, но не переписывают: `sources/`, `inbox/`, записи в watchcards.
- `publish/` трогает только человек (или агент по прямому поручению человека).
- Спекуляция всегда помечена (**Watch / Object of Attention**) — политика из WORKFLOW.md действует.
- Каждое утверждение — с первичной ссылкой; без ссылки — в inbox, не в sources.
- Автоматика фиксирует; **суждение остаётся за людьми и именованными персонами**.

## Управление

- Ритм, темы и остановка цикла — по слову издателя в основной сессии
  («останови обходы», «перейди на еженедельный ритм», «добавь тему X»).
- Долг очереди: в `publish/substack/queue/` ~25 готовых текстов при одном
  опубликованном. Первая редакционная задача — сверить очередь с реальным
  Substack и восстановить движение queue → published.
